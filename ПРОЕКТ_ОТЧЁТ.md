**Метод:** `save()` - **Строки 53-72** (СОЗДАНИЕ И ОБНОВЛЕНИЕ)

```java
@PostMapping("/books")  // Строка 53: POST запрос на /books (отправка формы)
public String save(@ModelAttribute Book book, RedirectAttributes ra) {
    // Строка 54: @ModelAttribute автоматически заполняет объект Book из формы
    
    // ВАЛИДАЦИЯ (строки 56-63):
    if (book.getTitle() == null || book.getTitle().trim().isEmpty()) {
        ra.addFlashAttribute("error", "Название книги обязательно");
        return "redirect:/books/new";  // Возврат на форму при ошибке
    }
    if (book.getAuthor() == null || book.getAuthor().trim().isEmpty()) {
        ra.addFlashAttribute("error", "Автор книги обязателен");
        return "redirect:/books/new";
    }
    
    // СОХРАНЕНИЕ (строка 65):
    repository.save(book);  // Сохраняет книгу в БД (создаёт, если id=null, обновляет если id!=null)
    
    // УСПЕШНОЕ СООБЩЕНИЕ (строка 66):
    ra.addFlashAttribute("success", 
        book.getId() != null ? "Книга обновлена" : "Книга добавлена");
    
    return "redirect:/books";  // Строка 67: Перенаправление на список книг
}
```

**Как работает создание:**
1. Пользователь заполняет форму на `/books/new`
2. Форма отправляется POST запросом на `/books`
3. Spring автоматически создаёт объект `Book` из данных формы (`@ModelAttribute`)
4. Проверяется валидность данных (название и автор обязательны)
5. Вызывается `repository.save(book)` - если `book.id == null`, создаётся новая запись
6. Пользователь перенаправляется на список книг с сообщением об успехе

```java
public String view(@PathVariable Long id, Model model) {
    // Строка 92: @PathVariable извлекает id из URL
    
    Optional<Book> b = repository.findById(id);  // Строка 93: Поиск книги по ID
    // repository.findById(id) - стандартный метод JpaRepository
    
    if (b.isPresent()) {  // Строка 94: Проверка, найдена ли книга
        model.addAttribute("book", b.get());  // Строка 95: Добавление книги в модель
        return "books/view";  // Строка 96: Возврат шаблона просмотра

    return "redirect:/books";  // Строка 98: Если книга не найдена - редирект на список
}
```

**Как работает:**
1. Из URL извлекается ID книги (например, `/books/5`)
2. Вызывается `repository.findById(5)` для поиска в БД
3. Если книга найдена - отображается страница просмотра
4. Если не найдена - перенаправление на список

```java
@PostMapping("/books")  // Тот же метод, что и для создания!

    ra.addFlashAttribute("success", 
        book.getId() != null ? "Книга обновлена" : "Книга добавлена");

```

**Как работает обновление:**
1. Пользователь нажимает "Редактировать" на книге с ID=5

3. Пользователь изменяет данные и отправляет форму
4. Форма отправляется POST на `/books` с заполненным `book.id = 5`
5. `repository.save(book)` видит, что `id != null`, и **ОБНОВЛЯЕТ** существующую запись
6. Пользователь видит сообщение "Книга обновлена"

**Важно:** Метод `save()` универсальный - он и создаёт, и обновляет в зависимости от наличия ID!

**Метод:** `register()` - **Строки 35-56** (ОБРАБОТКА РЕГИСТРАЦИИ)

```java
@PostMapping("/register")  // Строка 35: POST запрос при отправке формы
public String register(@ModelAttribute User user, RedirectAttributes ra) {
    // Строка 36: @ModelAttribute автоматически заполняет User из формы
    
    // ВАЛИДАЦИЯ ЛОГИНА (строки 37-40):
    if (user.getUsername() == null || user.getUsername().trim().isEmpty()) {
        ra.addFlashAttribute("error", "Логин обязателен");
        return "redirect:/register";  // Возврат на форму при ошибке
    }
    
    // ВАЛИДАЦИЯ ПАРОЛЯ (строки 41-44):
    if (user.getPassword() == null || user.getPassword().trim().isEmpty()) {
        ra.addFlashAttribute("error", "Пароль обязателен");
        return "redirect:/register";
    }
    
    // ПРОВЕРКА УНИКАЛЬНОСТИ (строки 46-49):
    if (userRepository.findByUsername(user.getUsername()).isPresent()) {
        ra.addFlashAttribute("error", "Пользователь с таким логином уже существует");
        return "redirect:/register";
    }
    
    // ШИФРОВАНИЕ ПАРОЛЯ (строка 51):
    user.setPassword(passwordEncoder.encode(user.getPassword()));
    // passwordEncoder - BCryptPasswordEncoder, шифрует пароль
    
    // УСТАНОВКА РОЛИ (строка 52):
    user.setRole("ROLE_USER");  // По умолчанию все пользователи - ROLE_USER
    
    // СОХРАНЕНИЕ В БД (строка 53):
    userRepository.save(user);  // Сохраняет нового пользователя
    
    // УСПЕШНОЕ СООБЩЕНИЕ (строка 54):
    ra.addFlashAttribute("success", "Регистрация успешна! Теперь вы можете войти.");
    
    return "redirect:/login";  // Строка 55: Перенаправление на страницу входа
}
```

**Как работает регистрация:**
1. Пользователь заполняет форму регистрации (логин, пароль)
2. Проверяется, что логин и пароль не пустые
3. Проверяется, что пользователь с таким логином не существует
4. Пароль шифруется через BCrypt (необратимое шифрование)
5. Устанавливается роль `ROLE_USER`
6. Пользователь сохраняется в БД
7. Перенаправление на страницу входа

**Безопасность:**
- Пароль никогда не хранится в открытом виде
- BCrypt - одностороннее шифрование (невозможно расшифровать)
- Уникальность логина гарантируется на уровне БД (`@Column(unique = true)`)

```java
@Configuration           // Строка 13: Класс конфигурации Spring
@EnableWebSecurity      // Строка 14: Включает Spring Security
public class SecurityConfig {

        return new BCryptPasswordEncoder();  // Строка 25: BCrypt для шифрования

        auth.setPasswordEncoder(passwordEncoder());      // Строка 32: Шифровальщик паролей
        return auth;
    }
    
    // НАСТРОЙКА ДОСТУПА (строки 36-54):
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .authenticationProvider(authenticationProvider())  // Строка 39: Использует наш провайдер
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/css/**", "/login", "/register").permitAll()  // Строка 41: Разрешено всем
                .anyRequest().authenticated()  // Строка 42: ВСЕ ОСТАЛЬНЫЕ требуют авторизации
            )
            .formLogin(form -> form
                .loginPage("/login")           // Строка 45: Страница входа
                .defaultSuccessUrl("/books", true)  // Строка 46: После входа → /books
                .permitAll()                  // Строка 47: Доступна всем
            )
            .logout(logout -> logout
                .logoutSuccessUrl("/login?logout").permitAll()  // Строка 50: После выхода → /login
            );
        
        return http.build();
    }
}
```

**Как работает безопасность:**

1. **Шифрование паролей (строки 23-26):**
   - BCryptPasswordEncoder шифрует пароли при регистрации
   - Сравнивает зашифрованные пароли при входе

2. **Провайдер аутентификации (строки 28-34):**
   - `DaoAuthenticationProvider` - проверяет логин/пароль
   - Использует `CustomUserDetailsService` для загрузки пользователя из БД
   - Использует `PasswordEncoder` для проверки пароля

3. **Правила доступа (строки 40-42):**
   - `/css/**`, `/login`, `/register` - доступны всем (без авторизации)
   - Все остальные страницы (`/books`, `/books/new` и т.д.) - требуют входа

4. **Форма входа (строки 44-47):**
   - Spring Security автоматически обрабатывает POST на `/login`
   - После успешного входа перенаправляет на `/books`

5. **Выход (строки 49-50):**
   - POST на `/logout` выходит из системы
   - Перенаправляет на `/login?logout`

### 👤 ЗАГРУЗКА ПОЛЬЗОВАТЕЛЕЙ

**Файл:** `src/main/java/com/example/library/service/CustomUserDetailsService.java`

**Назначение:** Загружает пользователя из БД для Spring Security

**Ключевые части кода:**

```java
@Service  // Строка 14: Spring Service
public class CustomUserDetailsService implements UserDetailsService {
    
    // ЗАГРУЗКА ПОЛЬЗОВАТЕЛЯ (строки 23-34):
    @Override
    public UserDetails loadUserByUsername(String username) throws UsernameNotFoundException {
        // Строка 24: Метод вызывается Spring Security при входе
        
        // ПОИСК В БД (строки 25-26):
        User user = userRepository.findByUsername(username)
                .orElseThrow(() -> new UsernameNotFoundException("User not found"));
        // Если пользователь не найден - выбрасывается исключение
        
        // СОЗДАНИЕ РОЛИ (строка 28):
        GrantedAuthority authority = new SimpleGrantedAuthority(user.getRole());
        // Преобразует строку "ROLE_USER" в объект роли
        
        // ВОЗВРАТ ДЛЯ SPRING SECURITY (строки 29-33):
        return new org.springframework.security.core.userdetails.User(
                user.getUsername(),    // Логин
                user.getPassword(),     // Зашифрованный пароль
                Collections.singleton(authority)  // Роль пользователя
        );
    }
}
```

**Как работает:**
1. При попытке входа Spring Security вызывает `loadUserByUsername(username)`
2. Метод ищет пользователя в БД по логину
3. Если найден - создаёт объект `UserDetails` с логином, паролем и ролью
4. Spring Security сравнивает введённый пароль с зашифрованным из БД
5. Если совпадают - вход успешен

```java
public interface BookRepository extends JpaRepository<Book, Long> {
    
    // ПОИСК ПО НАЗВАНИЮ, АВТОРУ ИЛИ ЖАНРУ (строки 11-12):
    @Query("SELECT b FROM Book b WHERE LOWER(b.title) LIKE LOWER(CONCAT('%', :query, '%')) OR LOWER(b.author) LIKE LOWER(CONCAT('%', :query, '%')) OR LOWER(b.genre) LIKE LOWER(CONCAT('%', :query, '%'))")

    // ФИЛЬТРАЦИЯ ПО ЖАНРУ И ГОДУ (строки 14-15):

}
```

**Разбор методов:**

1. **`searchBooks(String query)` - Строки 11-12:**

   - `:year IS NULL OR ...` - если год не указан, игнорируется
   - Можно фильтровать только по жанру, только по году, или по обоим

3. **`findAllGenres()` - Строки 17-18:**

### 🔎 ПОИСК ПО ТЕКСТУ

**Метод в Repository:** `searchBooks(String query)` - **Строки 11-12**

**SQL запрос (JPQL):**
```sql
SELECT b FROM Book b 
WHERE LOWER(b.title) LIKE LOWER(CONCAT('%', :query, '%')) 
   OR LOWER(b.author) LIKE LOWER(CONCAT('%', :query, '%')) 
   OR LOWER(b.genre) LIKE LOWER(CONCAT('%', :query, '%'))
```

**Как работает:**
1. Преобразует запрос и поля в нижний регистр (`LOWER()`)
2. Ищет совпадения в трёх полях: название, автор, жанр
3. `LIKE CONCAT('%', :query, '%')` означает "содержит"
   - Запрос "тол" найдёт "Толстой", "толстого", "ТОЛСТОЙ"
4. `OR` означает, что достаточно совпадения в любом поле

**Примеры:**
- Запрос "толстой" → найдёт книги автора "Толстой"
- Запрос "фантастика" → найдёт книги жанра "Фантастика"
- Запрос "война" → найдёт книги с "война" в названии

**Вызов в контроллере:**
```java
// Строка 28: BookController.list()
model.addAttribute("books", repository.searchBooks(search.trim()));
```

### 🎯 ФИЛЬТРАЦИЯ ПО ЖАНРУ И ГОДУ

**Метод в Repository:** `filterBooks(String genre, Integer year)` - **Строки 14-15**

**SQL запрос (JPQL):**
```sql
SELECT b FROM Book b 
WHERE (:genre IS NULL OR LOWER(b.genre) LIKE LOWER(CONCAT('%', :genre, '%'))) 
  AND (:year IS NULL OR b.year = :year)
```

**Как работает:**
1. **Фильтр по жанру:**
   - Если `genre` не указан (`IS NULL`) → игнорируется
   - Если указан → ищет частичное совпадение (как в поиске)
   - Пример: "фант" найдёт "Фантастика", "Научная фантастика"

2. **Фильтр по году:**
   - Если `year` не указан (`IS NULL`) → игнорируется
   - Если указан → точное совпадение (`=`)
   - Пример: `year=2020` найдёт только книги 2020 года

3. **Комбинация:**
   - `AND` означает, что должны выполняться оба условия
   - Можно указать только жанр, только год, или оба

**Примеры:**
- `/books?genre=Фантастика` → все книги жанра "Фантастика"
- `/books?year=2020` → все книги 2020 года
- `/books?genre=Классика&year=1967` → классика 1967 года

**Вызов в контроллере:**
```java
// Строка 31: BookController.list()
model.addAttribute("books", repository.filterBooks(genre, year));
```

### Разработчик 1 (CRUD):
- ✅ CREATE: `createForm()` + `save()` (id=null)
- ✅ READ: `list()` + `view()`
- ✅ UPDATE: `editForm()` + `save()` (id!=null)
- ✅ DELETE: `delete()`

### Разработчик 2 (Аутентификация):
- ✅ Регистрация: `registerForm()` + `register()`
- ✅ Вход: `login()` + Spring Security
- ✅ Выход: Spring Security logout
- ✅ Защита страниц: `SecurityConfig`
- ✅ Шифрование паролей: BCrypt

### Разработчик 3 (Поиск/Фильтры):
- ✅ Поиск по тексту: `searchBooks()`
- ✅ Фильтр по жанру: `filterBooks(genre, null)`
- ✅ Фильтр по году: `filterBooks(null, year)`
- ✅ Комбинированный фильтр: `filterBooks(genre, year)`
- ✅ Список жанров: `findAllGenres()`
